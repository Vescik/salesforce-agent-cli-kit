# Skill: Generate User Story Context

## Purpose

Generate clear User Story context from provided Description and Acceptance Criteria.

## Rules

- Base the context only on the provided Description and Acceptance Criteria.
- Do not invent missing requirements.
- Convert business description into Salesforce implementation context.
- List missing information under `## Questions / Clarifications`.

## Output Format

# User Story Context

## User Story

As a [user/persona],
I want to [goal],
so that [business value].

## Background

## Business Problem

## Expected Outcome

## Acceptance Criteria Summary

## Salesforce Impact

## Metadata Components Likely Involved

## Dependencies

## Assumptions

## Questions / Clarifications

## Testing Notes

## Output

Save output to:

`output/user-stories/user-story-context.md`
