# Admin Salesforce Concepts Glossary

## Object
A table-like container for records, such as Account, Contact, or a custom object.

## Field
A piece of information stored on a record, such as Status, Amount, or Email.

## Record
One saved item inside an object, like one Account or one Invoice.

## Lookup Relationship
A flexible link from one record to another. The child can usually exist without the parent.

## Master-Detail Relationship
A strong parent-child relationship where the child depends on the parent and may inherit sharing.

## Junction Object
A custom object used to connect two objects in a many-to-many relationship.

## Record Type
A way to show different business processes, picklist values, and layouts for the same object.

## Page Layout
Controls fields, buttons, and related lists in parts of the Salesforce interface.

## Lightning Record Page
The Lightning page users see for a record. It can include components, visibility rules, Dynamic Forms, and Dynamic Actions.

## Dynamic Forms
Lightning page feature that lets fields and sections appear based on conditions.

## Dynamic Actions
Lightning page feature that lets actions appear based on conditions.

## Related List
A list of child or related records shown on a parent record.

## Profile
A baseline access configuration for a user.

## Permission Set
An additional bundle of permissions assigned to users.

## Permission Set Group
A group of permission sets assigned together.

## Role Hierarchy
A sharing structure that can give managers or higher roles access to records owned below them.

## Sharing Rule
A rule that opens record access to users, roles, groups, or territories.

## Org-Wide Defaults
The baseline record sharing level for each object.

## Flow
A Salesforce automation tool that can guide users or automate record actions.

## Record-Triggered Flow
A Flow that runs when a record is created, updated, or deleted.

## Screen Flow
A Flow that shows screens and guides a user through steps.

## Validation Rule
A rule that blocks saving a record when the data does not meet business requirements.

## Approval Process
A formal review process that can lock records and request approvals.

## Apex Trigger
Developer-written code that runs when records change.

## Report Type
Defines which objects and records a report can use.

## Dashboard Running User
The user whose access controls what data a dashboard can show, unless dynamic dashboards are used.

## Field History Tracking
A feature that records changes to selected fields.

## Setup Audit Trail
A log of many configuration changes made in Setup.

## Debug Log
A technical log that can help developers investigate automation and code behavior.
